└───────────────────────────────┘

### 🔑 设计原则

1. **QML 页面层**

   - 负责布局和组合控件
   - 使用绑定/信号连接到 Model
   - 纯 UI，**不做业务处理**

2. **QML 控件层（公共组件）**

   - 封装可复用控件（滑块、按钮、滤镜面板）
   - 提供 `property`、`signal` 与外界通信
   - 只负责 UI 样式和轻量交互事件

3. **C++ Model 层**

   - 核心业务逻辑全部放这里
   - 性能敏感计算（RAW 解码、滤镜计算）
   - 提供属性/信号给 QML 绑定

---

这种设计思路的好处：

- **复用控件简单**：不同页面都可以直接用相同控件
- **UI 与业务解耦**：方便跨平台维护
- **性能优化明确**：重计算都在 C++，QML 只做渲染和事件
